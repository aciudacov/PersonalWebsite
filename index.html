Portfolio

Loadboard Scanner Bot

Architecture

.NET API as backend with puppeteer to retrieve platform tokens, UI served via Github Pages, Telegram bot as control

Tech Stack

.NET Telegram Bot API Puppeteer GitHub Pages JavaScript

Description

I built a real-time load monitoring system for truck dispatchers that aggregates multiple loadboards and delivers filtered alerts via Telegram, removing the need to manually track several platforms.

Since official APIs were limited or lacked search capabilities, I used the same internal endpoints as web clients and combined them with browser automation for authentication. The system polls platforms every 5–10 seconds and processes results through a filtering and deduplication pipeline based on user-defined criteria.

The backend is implemented in .NET with caching and optimizations to reduce redundant processing while maintaining low-latency delivery. The product evolved from a command-based Telegram bot into a system with an additional web UI for improved usability.

It runs across multiple instances and serves 100+ active users. I designed, built, and maintain the entire system end-to-end.
